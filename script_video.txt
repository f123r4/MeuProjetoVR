[CENA 1 — INTRODUÇÃO]  (~30 segundos)
Tela: Título do projeto / Unity aberto com a cena
--------------------------------------------------

"Olá! Meu nome é Filipe Mazon e esse é o meu projeto
final da Unidade 1 da Residência em TIC 29, trilha Web 3.0,
com a professora Ana Beatriz.

O projeto se chama 'Meu Primeiro Ambiente VR' e é uma
experiência de Realidade Virtual desenvolvida com
Unity 6 e o Meta XR SDK, o kit oficial da Meta para
criação de aplicativos para o Quest.

Vou mostrar o que foi desenvolvido, como funciona
e como o código foi organizado."


--------------------------------------------------
[CENA 2 — VISÃO GERAL DO AMBIENTE]  (~45 segundos)
Tela: Cena aberta no Unity, visão geral do cenário
--------------------------------------------------

"Aqui está a cena principal do projeto — scene1.unity.

O ambiente é composto por um plano de chão, uma iluminação
direcional e uma casa construída completamente por código,
usando um editor customizado chamado CasaBuilder —
um script de utilitário que gera todas as paredes,
o teto, a porta e até a maçaneta automaticamente
dentro do Unity Editor, sem precisar montar peça por peça.

No cenário também temos três objetos coletáveis:
- Uma Chave, que vale 10 pontos
- Um Cristal, que vale 20 pontos
- Uma Moeda, que vale 30 pontos

E um Botão Principal com o qual o jogador pode interagir."


--------------------------------------------------
[CENA 3 — DEMONSTRAÇÃO DO GAMEPLAY]  (~60 segundos)
Tela: Projeto rodando no Play Mode dentro do Unity
--------------------------------------------------

"Vou apertar o Play para mostrar o jogo funcionando
diretamente no Editor, sem precisar de headset.

[APERTAR PLAY]

O jogador se move com as teclas W, A, S, D ou
as setas do teclado.

[MOVER EM DIREÇÃO A UM OBJETO]

Quando me aproximo de um objeto coletável, ele é
coletado automaticamente por proximidade.
Repara que o HUD aqui em cima atualizou a pontuação
e o contador de objetos.

[MOVER EM DIREÇÃO AO BOTÃO]

Quando me aproximo do Botão Principal, ele muda de cor
para amarelo — isso é o efeito de 'hover'.
A mensagem no HUD também mudou, pedindo para pressionar E.

[PRESSIONAR A TECLA E]

Ao pressionar E, o botão fica verde e a mensagem confirma
a ativação. Tudo isso com feedback visual e texto em tempo real."


--------------------------------------------------
[CENA 4 — ARQUITETURA DO CÓDIGO (MVC)]  (~90 segundos)
Tela: VS Code ou Unity com a pasta Scripts aberta
--------------------------------------------------

"Agora vou falar sobre como o código foi organizado.
O projeto usa a arquitetura MVC — Model, View e Controller.

Começando pelo CONTROLLER:

O GameManager.cs é o controlador central.
Ele guarda a pontuação e o total de objetos.
Quando um objeto é coletado, ele chama
o método RegistrarColeta(), que incrementa os pontos
e avisa a View para atualizar o HUD.

O PlayerController.cs cuida da movimentação do jogador
e também detecta colisões por proximidade usando
OverlapSphere — uma esfera invisível ao redor do player
que detecta objetos próximos a cada frame.

O ObjetoColetavelController.cs é chamado quando o player
entra na esfera de coleta. Ele verifica se o objeto
já foi coletado — evitando dupla contagem — e chama
o GameManager para registrar os pontos.

O BotaoPrincipalController.cs gerencia os estados
do botão: hover ao se aproximar e ativação ao pressionar E.

Passando para as VIEWS:

A HUDView.cs controla a interface flutuante do jogador.
Ela segue a câmera usando o LateUpdate — ou seja,
ela sempre fica na frente do player, como um HUD de VR.

A ObjetoColetavelView.cs e BotaoPrincipalView.cs
controlam apenas o visual: animações e mudança de cor.

Essa separação deixa o código limpo: a lógica
não mistura com o visual, e cada arquivo tem
uma responsabilidade clara."


--------------------------------------------------
[CENA 5 — COMPATIBILIDADE COM META QUEST]  (~45 segundos)
Tela: Project Settings aberto, aba Android / XR
--------------------------------------------------

"O projeto foi configurado para rodar no Meta Quest.

O XR Plugin Management está com o plugin Oculus ativo
tanto para PC quanto para Android.

As configurações de build seguem os requisitos do Quest:
plataforma Android, API mínima 29, compressão ASTC,
backend de script IL2CPP e arquitetura ARM64.

Para testar sem o headset, o Meta XR Simulator pode ser
ativado em Window → XR → OpenXR → Meta XR Simulator.

Para fazer o build real, basta conectar o Quest via USB
com o Modo Desenvolvedor ativado e usar
File → Build Settings → Build and Run."


--------------------------------------------------
[CENA 6 — ENCERRAMENTO]  (~30 segundos)
Tela: Cena do Unity ou tela de título
--------------------------------------------------

"Esse foi o meu projeto 'Meu Primeiro Ambiente VR'.

O repositório está disponível no GitHub — o link

Usei Unity 6 com Meta XR SDK, organizei o código em MVC
com scripts dedicados para controller, view e lógica de jogo,
e construí o ambiente incluindo um editor customizado.

Obrigado pela atenção!"
